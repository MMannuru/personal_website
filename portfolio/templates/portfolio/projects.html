Some projects I've worked/been working on.

GameLogs - Video Game Tracker and Community Platform

Currently building. GameLogs allows users to find, track, and share video games with friends and the community. It provides a comprehensive system for managing game backlogs, creating reviews, and finding new games through personalized recommendations. Users can search for games and friends, view detailed game information, and keep up with the latest trends in gaming through an activity feed.

  • Technologies: React, Express/Node, PostgresSQL, PyTorch

Language Detection System with Kernelized Support Vector Machines

Developed a language detection system using Support Vector Machines (SVM) for rapid and accurate classification (95%) of languages. The model uses a TfidfVectorizer to extract character-level n-grams as features, enabling effective handling of diverse languages with minimal data. The pipeline integrates a LinearSVC classifier for efficient training and prediction. SVMs are powerful when operating in high-dimensional feature spaces, which makes them great for solving problems like this. The model is also incredibly fast and only takes 2-3 seconds for prediction.

  • Technologies: Python, Scikit-Learn

Neural Networks from Scratch to Predict House Prices in California

Built a fully connected neural network including non-linear activation functions, forward propagation, backpropagation, and gradient descent algorithms from scratch to classify the price of houses in California.

  • Technologies: Python, NumPy

SummarAI: A Smart Study Buddy that Summarizes Notes and Answers Questions!

I built this to learn how to use a new technology. SummarAI is an app that allows you to summarize PDFs and ask questions about them. The motivation behind this app was to help students study. When studying for classes, I found it greatly beneficial to first get an idea of what I'm going to learn. This can be applied to all PDFs. This app uses EvaDB, an AI database system that makes it easy to use machine learning models on SQL and vector databases. Here it is used to help summarize content from a user's given PDF (EvaDB's in-built support for BART is used in this app). SQLite is the default database engine that EvaDB connects to.

  • Technologies: Python, SQLite, EvaDB
